### 4. 补充会议号

**问题描述：**

失信主单状态为“发送SRM失败”，且日志表错误日志为“子分公司采委会上会纪要文号[company_summary_id]不能为空”。



**问题原因：**

2021年4月28日发版上线后，增加了采购支持中心分管领导审批节点，失信行为主单字段“采委会上会纪要文号”字段由采购支持中心分管领导批量填入，但428上线之前提报的单据审批流不经过采购支持中心分管领导，导致会议号、会议时间字段无法从功能中录入系统。



**问题处理：**

```sql

#更新
update t_bad_behavior set
company_summary_id = '物资公司物资与采购工作领导小组2021年第二十二次会议',company_summary_time = '2021-11-01'
```

### 5.供应商统代不能为空

**问题描述：**

失信主单状态为“发送SRM失败”，且日志表错误日志为“第2行：供应商统一社会信用代码[credit_no]不能为空”。



**问题原因：**

失信发送SRM时，SRM系统会校验e购传输的供应商名称、统代、主数据编码和SRM是否一致，如不一致，需双方核对三个信息，此保持主要是因为e购侧供应商信息表和失信子表确实统代信息导致。



**问题处理：**

```sql
#询价中心
INSERT INTO t_bad_behavior_supplier_bak1
SELECT '','actionview-1757',NOW(),'',a.* from t_bad_behavior_supplier a where supplier_name in (
'宜兴市海华环保设备有限公司','大连德源林机电设备有限公司');

update t_bad_behavior_supplier set credit_no='9132028262841310X4' where supplier_name='宜兴市海华环保设备有限公司';

update t_bad_behavior_supplier set credit_no='91210200691441007J' where supplier_name='大连德源林机电设备有限公司';

#会员中心
INSERT INTO t_supplier_info_bak1
SELECT '','actionview-1757',NOW(),'',a.* from t_supplier_info a where supplier_name in (
'宜兴市海华环保设备有限公司','大连德源林机电设备有限公司');
```

### 6. 供应商名称或统一社会信用代码与SRM系统不一致

**问题描述：**

失信主单状态为“发送SRM失败”，且日志表错误日志为“第 1  行：供应商名称或统一社会信用代码与SRM系统不一致”。



**问题原因：**

失信发送SRM时，SRM系统会校验e购传输的供应商名称、统代、主数据编码和SRM是否一致，如不一致，需双方核对三个信息，此错误主要是因为SRM统代信息不正确导致，需联系SRM实施顾问处理。



### 7. 供应商 0030820223 不存在

**问题描述：**

失信主单状态为“发送SRM失败”，且日志表错误日志为“供应商 0030820223 不存在”。



**问题原因：**

此问题主要是因为SRM系统中不存在该供应商导致，需先联系采购支持中心宋立娟到ERP系统中扩展，然后联系SRM实施顾问拉取该供应商即可。

### 9. 缺少标包编号或标包编号不存在

**问题描述：**

失信主单状态为“发送SRM失败”，且日志表错误日志为“缺少标包编号或标包编号不存在”。



**问题原因：**

2021年11月30日上线新增了标包编号字段，e购发送至SRM，由于SRM可能存在删除标包或采购中止后找不到标包，导致srm校验返回此错误。（已向SRM反馈，下一版上线后修改该问题，不做该校验）
